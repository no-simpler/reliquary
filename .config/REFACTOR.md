# Reliquary refactor backlog

Items intentionally deferred from the holistic audit on 2026-04-26. Each is sized to be picked up independently. Sorted roughly by impact.

## 1. Drift-proof shared aliases & helpers across shells

**Today.** Aliases and helper functions are duplicated in parallel POSIX (`*.sh`) and fish (`*.fish`) files with no enforcement of equivalence. Real divergence already crept in (e.g. `pyenv init -` was zsh-only — fixed in this audit; `dcex` quoting differs harmlessly between shells).

Affected files:
- `~/.config/shell/090-funcs.sh` ↔ `~/.config/fish/conf.d/090-funcs.fish` (`mcd`, `tca`)
- `~/.config/shell/100-aliases-{docker,git,yadm}.sh` ↔ `~/.config/fish/conf.d/100-aliases-{docker,git,yadm}.fish`
- `~/.config/shell/100-aliases.sh` ↔ `~/.config/fish/conf.d/100-aliases.fish`

**Options.**
- (a) Convention-only: add a `bin/check-shell-parity` script that diffs alias/function names between paired files and exits non-zero on mismatch; wire it into `up` or a yadm pre-commit hook.
- (b) Generator-based: define a single TOML/YAML alias spec, emit `*.sh` and `*.fish` from it. Eliminates drift entirely but adds a build step.

Recommend (a) first — minimal tooling, high signal.

## 2. Move shell rc files inward (user-stated wish)

**Today.** `.bashrc`, `.bash_profile`, `.zshrc`, `.zprofile`, `.hushlogin` live tracked at `$HOME` root. User wants them under `~/.config/`.

**Plan.**
- **zsh** is fully relocatable via `ZDOTDIR`. Set `ZDOTDIR=$HOME/.config/zsh` in a tiny `~/.zshenv` (which can't move — zsh's hardcoded discovery starts at `$HOME/.zshenv`). Move `.zshrc` and `.zshprofile` into `~/.config/zsh/`. Net: only `~/.zshenv` stays at root.
- **bash** has no `BASHDOTDIR` equivalent. Two options:
  - (a) Keep `~/.bashrc` and `~/.bash_profile` at $HOME as one-line forwarders that source `~/.config/bash/{bashrc,bash_profile}`.
  - (b) Symlink `~/.bashrc` → `~/.config/bash/bashrc` from a bootstrap snippet.
- Update `~/.config/yadm/snippets/` (likely a new `shared/` snippet) to install the bash forwarders/symlinks on new-machine setup.

## 3. Tracked-on-disk root dotfiles still untracked

`~/.zshenv` (one-line: `. "$HOME/.cargo/env"`) and `~/.profile` (same line) are auto-generated by rustup. Decide:
- Track as canonical (then rustup's append on new machines is a duplicate but harmless).
- Document as "leave to rustup regeneration" in `~/.github/README.md` bootstrap notes.

If item 2 above moves `.zshrc` inward via `ZDOTDIR`, `~/.zshenv` becomes load-bearing (sets `ZDOTDIR`) and MUST be tracked.

## 5. `yadm-wrapper` non-interactive accessibility

Today the wrapper-only subcommands (`verify`, `check`, `update`, `own`, `disown`) only work because `~/.config/bin/yadm-wrapper` is shell-aliased to `yadm` in interactive shells. Scripts and Claude Code's `Bash` tool have to call `~/.config/bin/yadm-wrapper <cmd>` directly.

Options:
- (a) Install a `yadm` shim earlier in `$PATH` than `/opt/homebrew/bin/yadm` (e.g. as `~/.config/bin/yadm` → wrapper). `~/.config/bin` is already on PATH ahead of brew. The wrapper would need to be careful not to recurse — call `/opt/homebrew/bin/yadm` explicitly (it already does via `$YADM_BIN`).
- (b) Rename the wrapper to a clearly distinct name like `y` or `dot` and document.
- (c) Status quo — accept the constraint, keep CLAUDE.md note current.

(a) is the cleanest; minor risk is a malformed shim breaking the very tool that manages itself.

## 8. Encryption pattern hygiene

`~/.config/yadm/encrypt` has 20 entries (after this audit) spanning multiple themes ("benefactor", "mm", "bt"). Per CLAUDE.md, names should be opaque — `mm`, `bt` already are. Consider:
- Splitting into themed sub-archives via `yadm encrypt -A` archive-name support (yadm supports multiple `archive` names).
- OR consolidating into fewer glob patterns (e.g. `.config/{shell,fish/conf.d}/150-benefactor.*` if a future yadm version supports brace globs in encrypt patterns).

## 9. Bash prompt parity gap

`~/.config/shell/050-prompt.{fish,zsh}` use oh-my-posh (theme: `dreamsofautonomy.toml`); `~/.config/shell/050-prompt.bash` rolls its own static `PS1`. Either:
- Port bash to oh-my-posh for consistency (oh-my-posh supports bash via `oh-my-posh init bash`).
- Or document bash as the deliberate "fallback shell" with no oh-my-posh integration.

## 10. `yadm doctor` self-check command

Single command in `~/.config/bin/` that runs:
- `yadm-wrapper check` (encrypted-files SHA256 drift)
- `yadm-wrapper verify` (full archive-vs-disk diff)
- Smoke-test fish/zsh/bash startup (`<shell> -ic 'echo ok'`)
- Run the parity check from item #1 once it exists
- Sanity-check `$PATH` for duplicates

Catches most of the issues from this audit automatically next time. Could also run as a post-checkout hook or as part of `up`.

## 11. `060-fzf.zsh` is zsh-only

Bash gets no fzf key bindings; fish has its own setup in `~/.config/fish/conf.d/fzf.fish` (Fisher plugin). Either:
- Add a `060-fzf.bash` mirror (bash also has `fzf --bash` since fzf 0.48).
- Or merge into `060-fzf.sh` using shell-detection to call the right `fzf --<shell>`.

## 12. `oh-my-posh` config could move into the (nominally) right place

`~/.config/oh-my-posh/dreamsofautonomy.toml` is the only theme. If you want machine-portable themes outside the home dir, this is fine as-is. If you want oh-my-posh to discover it via `POSH_THEMES_PATH`, update `040-env.*` accordingly.
